# 게임 이펙트(VFX) 작업지시서 — 코딩 에이전트(메인 개발 CLI)용

> ⚠ 이 파일만은 **이미지 생성용이 아니다.** 이펙트는 90%가 코드(Phaser 트윈/그래픽스)라서
> 메인 개발 CLI(클로드 코드)에 아래 지시문을 **한 건씩 복붙**해서 시킨다. 이미지가 필요한 나머지 10%
> (글로우/스파크/블랙홀 텍스처, 폭발 스프라이트시트)는 `10_fx_textures.md`의 프롬프트로 이미 준비되어 있다.
>
> 순서대로 진행 권장: WO-1 → WO-2가 체감 효과가 가장 크고, WO-3~6은 계열별 디테일, WO-7~8은 이미지가 준비된 뒤.

## WO-1. 타격감 공통 레이어 (셰이크·히트스톱·피격 플래시) ★최우선

```
BattleScene에 타격감 공통 레이어를 추가해줘. docs/design/art_vfx_plan.md §5.3 기준.

1) 화면 셰이크: this.cameras.main.shake() 사용. 강도 3단계 — light(일반 명중), medium(폭발/5×5),
   heavy(격파). 폭발과 격파에만 걸고 일반 단발 명중은 light도 생략 가능하게.
2) 히트스톱: 명중 확정 순간 this.tweens.timeScale과 this.time.timeScale을 잠깐 낮췄다 복구(60~90ms).
   격파와 T5 무기 명중에만 적용.
3) 피격 플래시: 맞은 유닛 스프라이트/컨테이너를 1~2프레임 백색 틴트 후 원복.

규칙: 모든 수치(셰이크 강도/지속, 히트스톱 ms, 플래시 ms)는 하드코딩 금지 —
defaultGameConfig에 combat.vfx.* 섹션을 신설하고 관제실에서 조정 가능하게.
컷인 토글처럼 settings에 이펙트 강도 off/low/full 옵션도 넣어줘(사진 민감성 대비).
완료 기준: 관제실 🧪 모의 전투에서 일반 명중/폭발/격파의 체감이 뚜렷이 구분된다.
```

## WO-2. 격파 연출 (유닛 파괴 시퀀스)

```
BattleScene에서 유닛 HP가 0이 되어 제거될 때의 격파 연출을 추가해줘.
weaponVfx.js에 playDestruction(scene, x, y) 함수를 새로 만들어서:
1) 작은 선행 폭발 1~2회(위치 랜덤 오프셋, 80ms 간격) → 2) 본 폭발(백색 섬광 + 확장 링,
   기존 playPlasmaExplosion 재활용 가능) → 3) 파편 6~10개가 사방으로 흩어지며 페이드 →
4) 유닛 스프라이트는 본 폭발 시점에 알파 페이드 + 살짝 스케일 다운으로 제거.
WO-1의 heavy 셰이크와 히트스톱을 여기서 호출. 연출 중에도 게임 로직은 블로킹하지 말 것
(파괴 판정은 즉시, 연출은 fire-and-forget — weaponVfx.js의 기존 패턴 유지).
완료 기준: 모의 전투에서 적 격파 시 "터졌다"는 느낌이 남. npm test 전부 통과.
```

## WO-3. Plasma 디테일 — 링별 시차 폭발 + 잔열 히트헤이즈

```
Plasma 계열 연출 두 가지를 다듬어줘 (docs/design/art_vfx_plan.md §5.2 Plasma 행).
1) 5×5 폭발(T3/T4): 현재 한 번에 터진다면, 중심 칸 → 안쪽 링 → 바깥 링 순서로
   60ms 간격의 시차를 두고 playPlasmaExplosion을 호출해 "퍼져나가는" 느낌으로.
   피해 적용 로직/판정은 건드리지 말고 연출 타이밍만.
2) 잔열 지대(T4 residual_heat 타일): 지속 중인 칸 위에 주황색 반투명 사각형이
   알파 0.15~0.35로 느리게 펄스하는 히트헤이즈를 추가. 타일 만료 시 함께 제거.
수치는 combat.vfx.plasma.*로. 완료 기준: 모의 전투 T4 헬파이어 버스트에서 확인.
```

## WO-4. Ion 디테일 — 잔류 스파크 + 스턴 링

```
Ion 계열 상태 연출을 추가해줘 (docs/design/art_vfx_plan.md §5.2 Ion 행).
1) 잔류 스파크: 이온 무기 명중 후 대상 주변에 0.5초간 작은 스파크 입자 3~5개가
   지지직거리다 사라짐 (playIonBolt의 지그재그 로직 축소 재활용).
2) 스턴 링: unitModifiers의 stun이 부착된 유닛 머리 위에 회전하는 담청색 스파크 링을 표시,
   스턴 소비/만료 시 제거. 기존 상태 라벨(📡 등)과 겹치지 않게 배치.
완료 기준: 모의 전투에서 T2 행동력 교란기로 스턴 부착 → 링 표시 → AP 소비 시 사라짐 확인.
```

## WO-5. Gravity 디테일 — 중력장 소용돌이 + 강제이동 이징

```
Gravity 계열 연출을 다듬어줘 (docs/design/art_vfx_plan.md §5.2 Gravity 행).
1) 중력장(gravity_well 필드효과) 지속 칸 위에 보라색 동심원이 천천히 안쪽으로
   수축-반복되는 소용돌이 연출 추가 (Graphics 원 2~3개를 스케일 트윈 루프, 알파 낮게).
   필드 만료 시 함께 제거.
2) 밀어내기/집결의 강제이동 트윈에 Back.easeIn 이징을 적용하고, 충돌 발생 시
   충돌 지점에 백색 플래시 + playHitImpact를 호출해 "부딪혔다"를 보여줄 것.
완료 기준: T1 그래비톤 램 밀어내기→충돌, T3 중력 우물 필드에서 확인.
```

## WO-6. Antimatter 디테일 — 소멸 분해 + 블랙홀 타일 연출

```
Antimatter 계열 연출을 다듬어줘 (docs/design/art_vfx_plan.md §5.2 Antimatter 행).
1) 완전 소멸(T5)로 유닛이 제거될 때: 일반 격파(WO-2)와 다르게, 유닛 스프라이트가
   마젠타 섬광 후 작은 사각 조각 8~12개로 흩어지며 각자 랜덤 방향으로 날아가 페이드
   (폭발이 아니라 "존재가 지워지는" 느낌 — 파편에 중력 없이 감속만).
2) 블랙홀 타일(black_hole 필드효과): 어두운 중심 원 + 주위를 도는 마젠타 입자 2~3개가
   나선으로 빨려 들어가는 루프 연출. 만료 시 한 번 수축 플래시 후 제거.
완료 기준: T5 완전 소멸 병기 처치, T4 미소 특이점 블랙홀 생성에서 확인.
```

## WO-7. 파티클 텍스처 연동 (이미지 준비 후 — `10_fx_textures.md` A 참고)

```
public/assets/fx_glow_soft.png(256), fx_spark.png(128)가 준비됐어.
weaponVfx.js의 입자/글로우 연출을 업그레이드해줘:
1) BattleScene preload에서 두 텍스처 로드 (없으면 기존 Graphics 방식으로 폴백 — 필수).
2) 원(circle)으로 그리던 글로우/파편 입자를 해당 텍스처 이미지 + ADD 블렌드 모드로 교체,
   setTint(familyColor(family))로 계열색 적용.
3) 타이밍/개수/움직임은 기존 그대로 — 껍데기만 교체.
완료 기준: 텍스처 있을 때 네온 느낌, 텍스처 삭제 시에도 에러 없이 기존 연출로 동작.
```

## WO-8. 폭발 스프라이트시트 연동 (이미지 준비 후 — `10_fx_textures.md` C 참고)

```
public/assets/fx_explosion_sheet.png (1024×1024, 4×4=16프레임, 프레임 256×256)가 준비됐어.
1) spritesheet로 로드해서 'explosion' 애니메이션 등록 (frameRate ~24, repeat 0).
2) WO-2의 playDestruction 본 폭발과 Plasma 5×5 중심 폭발에서, 시트가 로드되어 있으면
   스프라이트 애니메이션 재생(끝나면 destroy), 없으면 기존 프로시저럴 폭발로 폴백.
fx_annihilation_sheet.png도 같은 방식으로 WO-6의 T5 소멸에 연결.
완료 기준: 시트 유무 양쪽 모두 정상 동작.
```
