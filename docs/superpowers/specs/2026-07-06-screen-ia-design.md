# 화면 구조(IA) 재설계 — 메인맵 / 전투맵 / 장소맵 3계층

> 작성: 2026-07-06 · 상태: **사용자 승인 완료** · 관련: `docs/DESIGN_BIBLE.md`
> 배경: 현재 모든 화면(성단맵/함대/정비허브/행성관리/저장)이 평면 전환이며, 상점·연구·조합이
> "장소 없는 전역 메뉴"로 우주 어디서든 열린다. 이를 장소 기반 3계층 구조로 재편한다.

## 확정된 근본 결정

1. **시설 메뉴(상점·연구·조합·수리·건조)는 해당 장소에 도킹했을 때만 사용 가능.**
2. **모항 다이달로스는 별도 화면이 아니라 장소맵의 하나** — 시설 구성만 다름.
3. **함대 편성(편성·장비 재배치·성장 확인)은 전역** — 단 구매는 상점, 건조·수리는 조선소 도킹 필요.

## 1. 화면 지도

```
[타이틀]
   ▼
[메인맵 main] ─── 게임의 기본 화면 (성단맵)
   ├─ 적 성계 노드 진입 ──▶ [전투맵 battle] ─ 승리/후퇴 ─▶ 메인맵 복귀
   │                         (점령 성공 시 해당 노드가 "장소"로 전환)
   └─ 모항/점령지 도킹 ──▶ [장소맵 place(placeId)] ─ 출항 ─▶ 메인맵 복귀
                              └─ 시설 탭 (그 장소에 존재하는 시설만)
[엔딩 ending] / [게임오버 gameover]는 종단 화면 (기존 유지)
```

화면 종류는 3개(main/place/battle) + 타이틀/종단뿐이다. fleet/hub/save/planet 독립 화면은 폐지한다.

## 2. 메뉴 소속표

| 메뉴 | 소속 | 비고 |
|---|---|---|
| 함대 편성 | 전역 (상단바) | 장비 "재배치"는 전역, "구매"는 상점 도킹 필요 |
| 저장/설정 | 전역 (시스템 메뉴) | |
| 관제실 (F9) | 전역 (개발자 전용) | 출시 전 개발 도구 — 변경 없음 |
| 상점 | 장소맵 · 상점 시설 | 모항 / 상점 시설 있는 행성 / 비밀상점 |
| 연구 | 장소맵 · 연구소 시설 | 모항 / 연구 기지 지은 행성 |
| 조합/제작 | 장소맵 · 워크샵 시설 | |
| 함선 건조·수리 | 장소맵 · 조선소 시설 | ShipyardPanel은 후속 단계 |
| 건물 관리 | 장소맵 · 모든 장소 | 모항=본부 건물 4종(사령부/연구소/워크샵/조선소) 업그레이드, 행성=아웃포스트 |
| 전직 | 장소맵 · 모항 | 기존 설정 유지 |

**핵심 규칙: 시설 탭은 그 장소에 해당 시설이 존재할 때만 생성된다.**
- 모항: `systems.json`의 `facilities` 배열 활용 (repair, class_change, research, shipyard, shop, fleet — 이미 존재).
- 점령 행성: 지은 건물이 시설을 제공한다. `buildings.js` 건물 정의에 `providesFacility` 필드 추가
  (예: 연구 기지 → `research`, 상점/교역소 → `shop`).

## 3. 상단바 변경

- 유지: 로고, 턴/함대 수, 자원 HUD, ⚙관제실 버튼.
- 햄버거 메뉴: `성단 맵/함대 편성/정비 허브/행성 관리/저장·설정` → **`함대 편성/저장·설정`만 유지.**
- 메인맵 복귀는 장소맵의 "출항" 버튼이 담당. 전투 중 전역 메뉴 잠금은 현행 유지.

## 4. 코드 구조

### 내비게이션 상태
- `App.jsx`: `view` 문자열 → `{ screen: 'title'|'main'|'place'|'battle'|'ending'|'gameover', placeId: string|null }`.
- 전환 함수는 화면 지도의 화살표와 1:1 대응 (`enterPlace(placeId)`, `exitPlace()`, `enterBattle(nodeId)`, …).

### 신규 PlaceScreen (장소맵)
- `src/ui/screens/PlaceScreen.jsx` — `placeId`로 장소 정보(모항/행성)를 읽어 시설 목록 계산 → 시설 탭 렌더.
- 시설 패널은 파일 분리: `src/ui/facilities/ShopPanel.jsx`, `ResearchPanel.jsx`, `CraftPanel.jsx`,
  `BuildPanel.jsx`, (후속) `ShipyardPanel.jsx`. 파일당 한 시설.
- 기존 `MaintenanceHubScreen`(연구·상점·조합 탭)과 `PlanetManagementScreen`(건설)은 해체되어
  위 패널들로 흡수된 후 삭제한다.

### 도킹 판정
- 메인맵에서 함대가 모항/점령 노드 위에 있을 때 "입항" 버튼 노출 → `enterPlace(nodeId)`.
- 함대 위치는 기존 StrategyMapScreen의 이동 상태(현재 노드)를 사용한다.

### 데이터 변경 (최소)
- `buildings.js`: 건물 정의에 `providesFacility` 필드 추가.
- `systems.json`: 변경 없음 (모항 `facilities` 이미 존재).
- 저장 호환: 화면 상태는 저장 대상이 아니므로 (Zustand 스토어만 저장) 세이브 마이그레이션 불필요.

### 에러 처리
- 존재하지 않는 `placeId`로 진입 시도 → 메인맵으로 폴백 + 콘솔 경고.
- 시설 없는 장소의 시설 탭 직접 접근 불가 (탭 자체가 렌더되지 않음).

## 5. 구현 순서 (커밋 단위)

| # | 단계 | 내용 | 완료 기준 |
|---|---|---|---|
| 1 | 내비게이션 골격 | screen/placeId 전환, PlaceScreen 셸(빈 탭), 상단바 정리 | 메인맵→장소맵→출항 왕복 동작, 기존 전투 진입 무손상 |
| 2 | 시설 패널 이식 | 연구/상점/조합 → PlaceScreen 시설 패널로 이동, 정비 허브 제거 | 모항 도킹 시에만 상점·연구·조합 사용 가능 |
| 3 | 행성 건설 통합 | PlanetManagementScreen 흡수, 건물→providesFacility 연동 | 행성 도킹 → 건설 탭, 건물 지으면 시설 탭 등장 |
| 4 | 도킹 UX | 메인맵 입항 버튼, 장소별 시설 검증, 수동 QA | 체크리스트 통과 |

## 6. 테스트

- 기존 vitest 47개 회귀 없음 (`npm test`).
- 수동 QA 체크리스트:
  - 타이틀 → 새 게임 → 메인맵 진입.
  - 모항 위에서 "입항" → 장소맵 → 수리/전직/연구/조선/상점 탭 표시 → "출항" → 메인맵.
  - 모항이 아닌 우주 공간에서 상점/연구 접근 경로가 존재하지 않음 (햄버거에 없음).
  - 적 노드 → 전투 → 승리 → 메인맵 복귀 → 점령지 입항 → 건설 탭.
  - 행성에 연구 기지 건설 → 연구 탭 등장.
  - 함대 편성은 메인맵/장소맵 어디서든 열림. 전투 중 잠금.
  - F9 관제실 어디서든 열림 (개발 도구).
  - 저장 → 로드 → 메인맵 정상 복귀.
