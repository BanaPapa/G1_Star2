## 1. PRD — 제품 정의

### 1.1 한 줄 정의

**함대를 키우고 성단을 정복하는 턴제 SF 택티컬 SRPG.** 그리드 전술 전투(창세기전·XCOM 계열)의 깊이와 에이스 필살기 컷인의 임팩트를, 행성 개발·연구·5계열 무기 성장 시스템 위에 올린다.

### 1.2 플랫폼·출시 목표

- **1차 출시: Steam (Windows)** — Electron 패키징. 가격대 미정(§Phase 12에서 결정)
- 기술 스택 (확정): Vite + React(UI) + Phaser 3(전술 전투) + Zustand, 데이터 주도(JSON), localStorage 저장
- 이후 확장 후보: 모바일(Capacitor), 데모/Next Fest

### 1.3 타겟

- 1차: 창세기전·영걸전·고전 SRPG 향수가 있는 30~40대
- 2차: XCOM·전략 RPG·로그라이트 성장 게임을 즐기는 코어 전략 게이머

### 1.4 핵심 루프

```
성단맵 탐험 → 적 조우/별계 정복(전투맵) → 보상·노획·투항 함선 획득
→ 입항(장소맵): 건물·연구·상점·조합·수리 → 함대 강화 → 다음 별계
```

### 1.5 3대 훅 (출시 시 반드시 작동해야 할 재미)

1. **전투의 맛** — 5계열 무기의 서로 다른 전투 문법(선긋기/교란/폭발/중력/소멸) + 지형·태세·기함
2. **임팩트** — TP 필살기 컷인 + 턴 캐릭터 카드 연출 (§Phase 8에서 리디자인)
3. **성장 보상** — 전투마다 함대가 분명히 강해지고 새 선택지(무기 티어·건물·시설)가 열림

### 1.6 게임 구조 (확정 — DESIGN_BIBLE §2)

- **화면 3계층**: 메인맵(성단맵) / 장소맵(모항·점령 행성, 입항 시) / 전투맵
- **MVP 맵**: 5노드 — 다이달로스(모항)→아르카디아→오르페우스→크로노스→니힐(최종보스). 시나리오 1~4장과 1:1
- **함선 6종**: 건십/프리깃/디스트로이어/크루저(헬리온급)/배틀크루저/배틀십
- **무기 5계열 25종**: Laser/Ion/Plasma/Gravity/Antimatter × Tier 1~5
- **에이스 4명**: 카이 레인(열혈)/세라 노바(냉정)/밀라 카르토(정비광)/레이븐(영입 조건부, 전 보이드 에이스)
- 모든 수치는 관제실(F9)에서 실시간 조정 가능 — 밸런싱은 데이터 작업

### 1.7 성공 기준 (출시 판정)

- 전술 전투 1판 3~5분, 별계 1개 15~30분, 전체 완주 4~6시간
- 첫 필살기 컷인에서 "멋지다" 반응
- 새 게임 → 엔딩까지 크래시/진행 불가 0건
- Steam 심사 통과 + 스토어 페이지(스크린샷 5+, 트레일러 1)

## 3. FSD — 기능 명세 (시스템별 상세)

> 각 시스템의 "무엇이며, 어디에 구현되어 있고, 무엇이 남았나". 새 세션은 담당 Phase의 관련 섹션만 읽으면 된다.

### 3.1 화면 구조 / 내비게이션 ✅

- **명세**: 메인맵 → (입항) 장소맵 / (조우) 전투맵. 시설 메뉴는 도킹 시에만. 전역 메뉴는 함대 편성/저장·설정 오버레이(Esc 닫기). 관제실(F9)은 개발 도구로 전역.
- **구현**: `App.jsx`(nav={screen,placeId}), `PlaceScreen.jsx`, `ui/facilities/*`(Research/Shop/Craft/Build 패널), `placeFacilities.js`(건물→시설 파생), `TopStatusBar.jsx`
- **스펙**: `superpowers/specs/2026-07-06-screen-ia-design.md`
- **잔여**: 없음 (조선소/수리 패널 내용물은 S11·S12)

### 3.2 성단맵 (전략 레이어) ✅

- **명세**: 캔버스 실시간 맵. WASD/화살표 이동 + 클릭 선택 + 자동 항해/궤도 정찰. 적 순찰대(위협도별 색/크기, 어그로 추격), 별계 수호 보스(접촉=정복 전투), 맵 이벤트(상인/표류선), 채굴, 모항 안전지대. 함대 좌표는 `useProgressStore.fleetPos`로 영속(전투/입항 후 유지).
- **구현**: `StrategyMapScreen.jsx` (1,700줄 — **Phase 11에서 분할 리팩토링 후보**)
- **잔여**: 파일 분할(그리기/AI/조우 판정 분리), 별계 진입 이벤트 다양화(Phase 7)

### 3.3 전투 시스템 (전술 레이어) ✅ 코어 / ❌ 무기 메커니즘

- **명세 (전투 v1.0 — 구현 완료)**:
  - 전장 크기 = 연구 티어별 10×8 ~ 20×16 (`BATTLEFIELD_SIZE_BY_TIER`)
  - AP 기반 행동 (이동/공격/방어/경계/스킬). 방어·경계 태세 = 남은 AP 전부 소모
  - 명중률/회피율 계산 파이프라인 + Priority Resolver로 순서 조정 가능
  - 피해: Shield(관통% 분할)→Armor(피해 감소)→HP. Armor는 내구도 별도. Shield 전투 중 자동 회복 없음, 전투 간 이월
  - 기함: 세력별 최강 유닛, 격파 시 상대 명중 페널티. 후퇴/교섭은 기함 판정(실패 시 기함 AP 0)
  - 투항: 격파 시 확률(기본 20%) 함선 포획 → 로스터 추가
  - Field Effect: 지뢰/포탈/잔열/중력장/블랙홀 (진입 피해·주기 피해·이동 비용)
  - 상태이상 시스템 없음 — 무기 효과는 **Unit Modifier**로 구현 예정 (DESIGN_BIBLE §3-6)
- **구현**: `game/scenes/BattleScene.js`(2,000줄+ — 분할 후보), `core/combatMath/*`(테스트 29개), `game/systems/terrain.js`, `core/ai.js`
- **잔여**: §Phase 4 전체 (무기 계열 메커니즘), 필살기 피해 파이프라인 통합(구 모델이 Shield 우회 — STAGE3 §4-1), 경계 태세 다중 턴 옵션

### 3.4 무기 5계열 ✅ 데이터 / ❌ 메커니즘 — **다음 작업 (Phase 4)**

- **명세**: `design/weapons_master_plan.md` (계열 문법·판정 규칙·보스 예외), `design/weapons_asset_table.md` (수치)
- **데이터**: `items.json` — 25종 완료. 현행 호환 필드(mods.atk/apCost/rangeBonus/pierce) + 설계 필드(range/accuracy/areaRadius/area/cooldown, 4단계에서 연결)
- **핵심 구현 요구사항** (master_plan §8):
  - AP 행동 태그 (MOVE/ATTACK/GUARD/SKILL) — Antimatter 1단계용
  - Unit Modifier 카테고리 (명중/이동/장갑 디버프 — 관제실 조정 가능)

## 6. 리스크·원칙 메모

| 리스크 | 대응 |
|---|---|
| 1인 개발 물량 (특히 아트 A1~A9) | 폴백 시스템 덕에 아트는 전 구간 병렬 가능. 이미지 생성 AI 파이프라인(§4.1) 활용. "플레이 가능 우선, 아트는 교체" |

| 밸런스 붕괴 | 모든 수치가 관제실 → 플레이하며 조정 → Export 반영. 수치 하드코딩 금지 원칙 유지 |
| 스코프 크리프 | DESIGN_BIBLE의 "향후 확장 요소"(7별 공개, 멀티엔딩 등)는 출시 후. 이 문서에 없는 기능은 기본 거절 |
| 세이브 호환성 | Phase 11-1 이전의 세이브는 깨질 수 있음(개발 중 허용). 출시 후엔 마이그레이션 필수 |
