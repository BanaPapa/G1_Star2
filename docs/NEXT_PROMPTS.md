# NEXT_PROMPTS — 후행작업 프롬프트 큐

> **용도**: 대화 세션이 바뀌어도 작업이 이어지도록, 남은 작업을 "복붙 가능한 프롬프트" 형태로 쌓아두는 문서.
> 새 세션은 이 파일 하나만 읽으면 된다 (CLAUDE.md가 자동으로 안내함).
>
> **운영 규칙** (코딩 에이전트용):
> 1. 사용자가 "다음 진행"이라고 하면 이 문서의 **대기 목록 맨 위 항목부터** 실행한다. 단, `⏸ 트리거` 조건이 안 갖춰진 항목은 건너뛴다.
> 2. 항목을 완료하면 **그 항목을 이 문서에서 삭제**하고, MASTER_PLAN 체크박스를 갱신한 뒤 함께 커밋한다.
> 3. 작업 중 새 후행작업이 생기면 아래 형식(트리거/프롬프트/완료 기준)으로 이 문서에 추가한다.
> 4. "현재 상황" 절은 문서를 갱신할 때마다 날짜와 함께 최신화한다.

## 대기 목록 (위에서부터 순서대로)

### 1. WO-7 — 파티클 텍스처 연동

**⏸ 트리거**: `public/assets/fx_glow_soft.png`(256), `fx_spark.png`(128)가 존재할 때.

```
public/assets/fx_glow_soft.png(256), fx_spark.png(128)가 준비됐어.
weaponVfx.js의 입자/글로우 연출을 업그레이드해줘:
1) BattleScene preload에서 두 텍스처 로드 (없으면 기존 Graphics 방식으로 폴백 — 필수).
2) 원(circle)으로 그리던 글로우/파편 입자를 해당 텍스처 이미지 + ADD 블렌드 모드로 교체,
   setTint(familyColor(family))로 계열색 적용.
3) 타이밍/개수/움직임은 기존 그대로 — 껍데기만 교체.
완료 기준: 텍스처 있을 때 네온 느낌, 텍스처 삭제 시에도 에러 없이 기존 연출로 동작.
```

**완료 기준**: 모의 전투에서 확인 + `npm test` 통과. 완료 시 `11_vfx_workorders.md` 상태 목록과 `art_vfx_plan.md` §1 E3 체크박스 갱신.

### 2. WO-8 — 폭발 스프라이트시트 연동

**⏸ 트리거**: `public/assets/fx_explosion_sheet.png`(1024×1024, 4×4 프레임)가 존재할 때. (`fx_annihilation_sheet.png`도 있으면 함께)

```
public/assets/fx_explosion_sheet.png (1024×1024, 4×4=16프레임, 프레임 256×256)가 준비됐어.
1) spritesheet로 로드해서 'explosion' 애니메이션 등록 (frameRate ~24, repeat 0).
2) WO-2의 playDestruction 본 폭발과 Plasma 5×5 중심 폭발에서, 시트가 로드되어 있으면
   스프라이트 애니메이션 재생(끝나면 destroy), 없으면 기존 프로시저럴 폭발로 폴백.
fx_annihilation_sheet.png도 같은 방식으로 WO-6의 T5 소멸에 연결.
완료 기준: 시트 유무 양쪽 모두 정상 동작.
```

**완료 기준**: WO-7과 동일 (모의 전투 확인, 테스트 통과, 문서 갱신 — E4 체크박스).

### 3. 아트 에셋 일괄 반영 QA

**⏸ 트리거**: 아트 세션이 `public/assets/`에 이미지(함선/무기/건물/배경/초상화 등)를 배치한 뒤.

```
public/assets/에 아트 세션이 만든 이미지들이 들어왔어. 게임 전체를 돌면서
새 에셋이 실제로 표시되는지 QA해줘:
1) 에셋 파일명이 코드가 기대하는 키(AssetImage assetKey, ships.json sprite,
   docs/design/image_prompts/ 각 파일의 파일명 규격)와 일치하는지 대조 — 불일치는 표로 보고.
2) 타이틀/성단맵/입항 화면/함대 편성/전투에서 ? 폴백이나 이모지가 남아 있는 곳 스크린샷.
3) 콘솔 404/로드 에러 없는지 확인.
누락·불일치는 수정하지 말고 먼저 보고할 것 (파일명 변경은 아트 세션 쪽에서 할 수도 있음).
```

**완료 기준**: 반영/누락 현황 보고서. 이후 필요 시 리네임·코드 수정은 별도 지시로.

### 4. Phase 6 시작 — 캐릭터·스토리 통합

**⏸ 트리거**: 위 1~3(아트·이펙트 반영)이 끝난 뒤. 사용자가 "다음 진행"이라고 하면 시작.

```
Phase 6를 시작하자. docs/MASTER_PLAN.md의 Phase 6 목록 순서대로.
첫 항목 6-1(고대 문명 정식 네이밍)은 사용자 결정이 필요하니, 시나리오 문서
(docs/design/scenario_main.md)의 톤에 맞는 네이밍 후보 3~5개를 근거와 함께
제안하고 내 선택을 받은 뒤 문서 일괄 반영해줘. 그 다음 6-2 대사 이벤트
시스템(EventModal 확장 + story.json 형식 설계)으로 진행.
```

**완료 기준**: 항목별 MASTER_PLAN 체크 + 커밋 (기존 Phase 진행 방식과 동일).
