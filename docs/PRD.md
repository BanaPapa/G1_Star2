# 7★ STAR — 제품 요구사항 정의서 (PRD) v1.0

> 작성: 프로젝트 리드 | 상태: MVP 확정 | 동반 문서: `dev_plan_guide.md`(개발 가이드), `game_data_dictionary.md`(데이터 사전)
>
> 본 PRD는 기존 첨부 문서들을 **참고만** 하고 백지에서 재정의했습니다. "무엇을, 왜" 만드는지를 정의하며, "어떻게"는 개발 가이드, "무슨 데이터로"는 데이터 사전이 담당합니다.

## 1. 개요

**한 줄 정의:** 함대를 키우고 은하를 정복하는 **턴제 SF 택티컬 SRPG**. 그리드 전술 전투의 깊이(창세기전 2 계열)와 필살기 컷인의 임팩트를, 깊은 성장·개발·수집 시스템 위에 올린다.

**장르:** 싱글플레이 턴제 전략 시뮬레이션 / 택티컬 RPG (전략 레이어 + 전술 레이어).

**핵심 판타지:** *"보잘것없는 함대로 시작해, 키우고 개발하고 동료를 모아, 압도적인 적을 전술과 필살기로 격파하는 사령관"*.

**플랫폼:** 웹 우선(브라우저) → 데스크톱(Electron/Steam)·모바일(Capacitor/Play) 확장.

---

## 2. 타겟 사용자

- **1차:** 창세기전·영걸전·Heroes of M&M·고전 SRPG에 향수가 있는 30~40대.
- **2차:** XCOM·전략 RPG·로그라이트 성장형 게임을 즐기는 코어 전략 게이머.
- **공통 동기:** "키우는 맛", "전술로 이기는 쾌감", "수집·발견", "임팩트 있는 연출".

---

## 3. 핵심 경험 — 왜 중독되는가

모든 재미의 척추는 **성장성**이다. 한 사이클마다 (1) 눈에 보이는 성장, (2) 놓치면 아까운 것(FOMO), (3) 새로 열리는 것(해금)이 동시에 존재해야 한다.

**핵심 루프:**
```
은하 탐험 → 별계 진입(이벤트/전투) → 그리드 전술 전투(위치·상성 → TP 충전 → 필살기 임팩트)
→ 보상·노획 → 정비(육성·전직·연구·조합·영입·상점) → 별계 정복·행성 개발 → 다음 별계
```

**3대 훅(MVP에서 반드시 작동해야 할 것):**
1. **전투의 맛** — 함선 상성·지형·명중/회피가 의미 있는 선택을 만든다(파랜드 택틱스 계열).
2. **임팩트** — TP를 모아 터뜨리는 에이스 필살기 컷인이 "와" 소리를 만든다(창세기전 2·용의 기사 2 계열).
3. **성장 보상** — 한 전투가 끝날 때마다 함대가 분명히 강해지고, 새 선택지가 열린다.

상세 시스템 설계는 `7star_systems_world_design.md`(설계서)를 참조.

## 4. ★ MVP 범위 정의 (확정)

> 원칙: **수직 슬라이스**. "넓게"가 아니라 "끝까지 한 덩어리". 세계관 비전은 크되, 첫 빌드는 1개 성단으로 한정한다.

### 4.1 맵 범위 — 1개 성단, 9개 노드

| 노드 | 역할 |
|------|------|
| 모항 (다이달로스 정거장) | 허브: 수리·전직·연구·조선·상점·함대 편성 |
| 정복 별계 ×7 (= 일곱 개의 별) | 미션 별계. 각자 고유 테마·지형·적·보상·히든·개발 해금 |
| 보스 별계 ×1 (심연의 눈) | 성단 보스전 → MVP 엔딩 + 다음 은하 떡밥 |

별계 상세(테마/적/보상/히든/개발)는 데이터 사전 2장 참조.

### 4.2 MVP 포함 콘텐츠 (수량 확정)

| 요소 | MVP 수량 |
|------|----------|
| 플레이어 함선 클래스 | 5종 (시작 3종 + 진행 해금 2종) |
| 에이스(지휘관) | 3명 + 영입 가능 적 에이스 1명 |
| 적 함선 | 5종(아군 미러 변형) + 고유 2종 + 미니보스 1 + 성단 보스 1 |
| 스킬·필살기 | 약 10종 |
| 자원 | 4종 (스텔라크레딧·티타늄·에너지크리스탈·다크매터) |
| 아이템 | 약 18종 (무기·모듈·소모품 + 히든 유니크 3) |
| 연구·개발 노드 | 약 8개 |
| 상점 | 모항 상점 + 비밀상점 1 + 떠돌이 상인 이벤트 |
| 이벤트 | 전투/자원/특별/정찰/평화 (개정판, 데이터 사전 11장) |

### 4.3 기능 요구사항 (MoSCoW)

**Must (MVP 필수)**
- 은하 전략맵 이동·별계 진입.
- 그리드 전술 전투: 이동·통상공격·스킬·필살기(TP)·방어, 함선 상성·명중/회피·지형 엄폐.
- 필살기 컷인 연출(정적 일러+이펙트+화면연출+사운드).
- 함선/에이스 성장·전직.
- 기본 연구·간단 조합(모듈 결합 1단계).
- 출격 전 상점(장비 구매·장착).
- 자원·기본 채굴.
- 별계 정복·행성 기본 개발(고유 해금 1~2개).
- 히든 유니크 소수(놓치면 못 얻음).
- 저장/불러오기.

**Should (가능하면)**
- 적 에이스 영입(조건부) 1건.
- 비밀상점 1곳.
- 떠돌이 상인 랜덤 조우.
- 이벤트 에피소드(감성 소품) 약간.

**Could (여유 시)**
- 광산 고갈·확장 압박.
- 연쇄 이벤트.
- 함선 조합 시너지 버프.

**Won't (MVP 비범위 — 향후)**
- 다중 은하(2~5 성단), 풀 조합 트리, 세력 외교, 수집 도감, 분기 서사, 멀티플레이어, 최종 보스 "The End of Time".

## 5. 게임 시스템 요약 (레이어)

- **전략 레이어:** 성단 맵 탐험, 별계 정복, 자원·광산 확보, 모항/행성 개발, 함대 편성, 연구·개발, 영입.
- **전술 레이어:** 그리드 턴제 전투. 위치·거리·지형·엄폐·상성·지휘관 능력·TP 필살기.
- **성장·개발:** 레벨·전직·필살기 해금 / 노획→연구→조합·제작(XCOM·창세기전1 계열).
- **경제·상점:** 자원 채집·전투 보상 → 상점·제작 소비(에어리어 88 계열 출격 전 구매).
- **수집·발견:** 맵 히든 유니크(용의 기사 2 계열), 비밀상점, 떠돌이 상인.
- **감성·연출:** 이벤트 소품·캐릭터 개성·필살기 연출(어스토니시아/테일즈위버 계열).

---

## 6. 성공 기준 (MVP 플레이테스트 목표)

- 전술 전투 1판 **3~5분**, 별계 1개 **15~30분**, MVP 완주 **3~5시간**.
- 플레이테스터의 **첫 필살기 컷인 반응**이 긍정적("멋지다/시원하다").
- "다음 별계로 가고 싶다"는 진행 욕구 발생(이탈 지점 ≥ 별계 3 이후).
- 함선 상성·지형이 "그냥 강한 함선 도배"가 아니라 **배치 고민**을 유발.
- 치명적 버그 0, 저장/로드 안정.

---

## 7. 제약·전제

- **기술:** 웹 스택 유지(React UI + PixiJS/Phaser 전투). 게임 엔진(Godot/Unity) 미사용. 데이터 주도 설계(JSON). 상세는 개발 가이드.
- **제작 방식:** 바이브코딩(AI 코드 생성) — 모듈 단위로 분할 요청. 상세는 개발 가이드 5장.
- **아트:** 그록 이미진 등으로 PNG 생성 → 후처리 → assetMap 연동. 엔진 독립.
- **법무(클린룸):** 레퍼런스 게임들의 **메커니즘·장르만 차용**. 그래픽·음악·스토리·캐릭터·이름·코드·상표 일절 미사용. 모든 고유명·아트·서사는 자체 제작. 상업화 본격화 시 IP 변호사 검토.
