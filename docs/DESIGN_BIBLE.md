## 2. 확정 사항 (양 계보 일치 + 구현 완료)

1. **기술 스택**: Vite + React(UI) + Phaser 3(전술 전투) + Zustand, 데이터 주도(JSON), localStorage 저장.
2. **장르 골격**: 전략 레이어(성계맵·개발·연구) + 전술 레이어(그리드 턴제 AP 전투).
3. **MVP 맵**: 5노드 — 다이달로스(모항) → 아르카디아 → 오르페우스 → 크로노스 → 니힐(보스). 시나리오 1~4장과 1:1 대응.
4. **플레이어 함선 6종**: 건십/프리깃/디스트로이어/크루저(헬리온급)/배틀크루저/(6번함급 — §3-3).
5. **전투 v1.0 규칙** (구현 완료): 연구 티어별 전장 크기(10×8 ~ 20×16), AP 태그 행동, 방어/경계 태세는 남은 AP 전부 소모, Shield→Armor→HP 파이프라인, Armor=피해 감소 방어력, Shield 전투 중 자동 회복 없음, 기함 중심 후퇴/교섭(실패 시 기함 AP 0), 투항 함선 획득, 아군 격파 시 함대에서 완전 삭제, Field Effect(지뢰/포탈/잔열/중력장/블랙홀).
6. **모든 수치는 관제실(F9)에서 조정 가능해야 한다** — 코드에 게임 수치 하드코딩 금지, `defaultGameConfig.js` + override가 단일 출처. Priority Resolver로 규칙 우선순위 드래그 조정.
7. **충돌 처리 원칙**: 기획 간 충돌이 보이면 삭제/임의 수정하지 않고 **관제실 토글 옵션**으로 설계한다.
8. **세력 컬러**: 플레이어 = 헬리온 연방 (화이트 `#F2F4F8` / 레드 `#FF3B3B` / 시안 `#00E0FF`).

## 3. 충돌 결정 기록

> 상태: ✅ 확정 / 🟡 규칙에 의해 자동 해결 / ❌ 미결

### 3-1. ✅ 무기 체계 — **5계열 25종 전면 도입** (2026-07-06 확정)
- `items.json`의 구식 무기 7종을 `wpn_*` 25종으로 교체하고, 계열별 고유 메커니즘
  (Laser 관통/굴절 → Ion 교란 → Plasma 폭발/잔열 → Gravity 강제이동/중력장 → Antimatter 블랙홀/소멸)을
  단계적으로 구현한다. 기준 문서: `design/weapons_master_plan.md`, 수치안: `design/weapons_asset_table.md`.

### 3-2. ✅ 에이스/TP/필살기/컷인 — **유지 + 시나리오에 통합** (2026-07-06 확정)
- 시스템(TP, 필살기, CutinManager)은 그대로 유지한다.
- 에이스 4명(kai, sera, mila, raven)을 메인 시나리오의 등장인물로 편입한다.
  → 로드맵 5단계 "캐릭터·스토리 연결"에서 각 에이스를 성계 1~4장 구조에 배치.

### 3-3. ✅ 6번째 함급 명칭 — **배틀십 (Battleship)** (2026-07-06 확정)
- 코드(`ships.json`)와 헬리온 연방 시트 표기를 그대로 사용. "Dreadnought" 표기는 폐기.

### 3-4. ✅ "일곱 개의 별"은 캐논 아님 — **참고용 컨셉이었음** (2026-07-06 확정)
- "일곱 개의 별"이라는 명칭·설정은 기획 시 컨셉 설명용 참고였을 뿐, 실제 게임 스토리와 무관하다.
- 유지되는 것: 몰락한 고대 문명 → 다이달로스 정거장 발견 → AI 흑막이라는 이야기 구조, 5성계 진행.
- **고대 문명의 정식 명칭은 미정** — 캐릭터·스토리 작업(로드맵 5단계) 때 네이밍한다.
- 게임명 "7★ STAR"도 이 결정과 함께 재검토 대상(가칭 유지).

### 3-5. 🟡 무기 쿨타임 — 전투 v1.0 "쿨타임 비활성화" vs 에셋표의 쿨타임 수치
- §2-7 원칙 적용: **관제실 토글로 구현** (`config.combat.cooldownEnabled`, 기본값 = 전투 v1.0에 따라 `false`). 에셋표의 쿨타임 수치는 데이터에 보존하되 토글 OFF 시 무시.

### 3-6. 🟡 상태이상 — 전투 v1.0 "상태이상 삭제" vs 무기 25종의 디버프 효과
- 해석 확정: 전투 v1.0이 삭제한 것은 **범용 상태이상 시스템**(독/빙결 같은 별도 레이어)이고, 무기 25종의 효과는 **무기 고유 효과(Weapon Effect)** 로 분류한다.
  - 지형에 남는 것(잔열/중력장/블랙홀) = Field Effect — 이미 구현됨.
  - 유닛에 붙는 것(명중 감소, 이동 제한, 장갑 약화 등) = Unit Modifier — 25종 구현 시 신규 카테고리로 추가, 수치·지속시간은 관제실에서 조정 가능하게.
