# START HERE — 클로드 코드 인수인계 안내서

> 이 파일은 **사람용 안내서**입니다. 클로드 코드는 이 대화의 맥락을 모르므로, 아래 "복붙용 첫 프롬프트"를 그대로 붙여 넣어 시작하세요.

---

## 1. 클로드 코드에 넘길 파일

새 빈 폴더(예: `7star-game/`)를 만들고, 아래를 넣은 뒤 클로드 코드를 그 폴더에서 여세요.

**문서 (docs/ 폴더에)**
- `PRD.md` — 무엇을·왜 (MVP 범위)
- `dev_plan_guide.md` — 모듈(MOD-0~13)·DoD·바이브코딩 가이드 ★가장 중요
- `game_data_dictionary.md` — 데이터 사전(사람이 읽는 버전)
- `7star_systems_world_design.md` — 시스템·세계관 "왜"(권장)

**데이터 (data/ 폴더에 — 그대로 src/data로 쓰임)**
- `ships.json` `enemies.json` `aces.json` `skills.json` `resources.json`
- `items.json` `research.json` `systems.json` `events.json` `shops.json`

> JSON은 이미 만들어져 있으니 클로드 코드는 **변환 없이 바로** 코드에 씁니다.

---

## 2. 복붙용 첫 프롬프트

아래를 그대로 클로드 코드 첫 메시지로 붙여 넣으세요.

```
나는 코딩 비전문가이고, 너(클로드 코드)와 바이브코딩으로 게임을 만들 거야.
너는 이전 대화 맥락이 없으니 아래 배경부터 알려줄게.

[프로젝트]
7★ STAR — 턴제 SF 택티컬 SRPG(우주 전략 시뮬레이션). 함대를 키워
은하를 정복하고, 그리드 전술 전투 + 에이스 필살기 컷인 연출이 핵심.
참고로 기존에 다른 코드가 있었지만 전부 버리고 백지에서 새로 시작한다.

[기술 스택 — 확정]
Vite + React(메뉴/전략맵 UI) + Phaser 3(전술 전투·컷인) + Zustand(상태),
데이터 주도(JSON), 저장은 localStorage. 배포는 추후 Electron/Capacitor.
게임 엔진(Godot/Unity)은 쓰지 않는다.

[자료]
docs/ 에 설계 문서 4개, data/ 에 게임 데이터 JSON 10개가 있다.
- 가장 중요한 건 docs/dev_plan_guide.md 다. 거기에 MOD-0~MOD-13 모듈과
  각 모듈의 '완료 정의(DoD)', 폴더 구조(1장)가 있다.
- docs/PRD.md 는 MVP 범위(1성단·9노드), data/*.json 은 함선·적·에이스·
  스킬·자원·아이템·연구·별계·이벤트·상점 데이터다.

[작업 규칙 — 꼭 지켜줘]
1. 먼저 docs 4개와 data JSON을 읽고, 이해한 내용을 짧게 요약해줘.
2. 그다음 dev_plan_guide.md 1장의 폴더 구조대로 빈 Vite 프로젝트를 셋업하고,
   data/*.json 을 src/data 로 옮겨 로드하는 MOD-0 까지만 해줘.
3. 한 번에 한 모듈씩만. 내가 "다음"이라고 하기 전엔 다음 모듈로 넘어가지 마.
4. 각 모듈은 dev_plan_guide.md 의 그 모듈 'DoD'를 충족하면 끝난 거야.
5. 없는 이미지는 이모지나 도형 플레이스홀더로 대체해(아트는 나중에 교체).
6. 수치는 data JSON 값을 그대로 써. 코드에 하드코딩하지 마.

이해 요약 → MOD-0 셋업, 이 순서로 시작해줘.
```

## 3. 그다음 흐름

- 모듈이 끝나면 직접 `npm run dev`로 실행 → 문제 있으면 **에러 메시지를 복붙**하고 "이 에러 고쳐줘".
- 잘 되면 **"다음"** → 클로드 코드가 dev_plan_guide.md 순서대로 MOD-1로 진행.
- 추천 진행 순서(가이드 3장): M0 → M1(전투 코어) → M2(필살기 컷인) → … → M6(보스·완주).
- 모듈 3~4개마다 **"전체 구조 정리·중복 제거해줘"** 한 번 요청.
- 막히면 **"현재까지 구현된 구조를 요약해줘"**로 컨텍스트를 다시 맞춰줘.
